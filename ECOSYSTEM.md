# RA 생태계 지도 (ECOSYSTEM)

> 의료기기 인허가(RA) 생태계를 구성하는 모든 조각과 그 관계를 명문화한 문서.
> **이 문서는 모든 개별 프로젝트 레포에 동일하게 주입된다.** 어느 레포도 상위가 아니며, 이 지도도 특정 조각에 종속되지 않는다.
> 1부(헌장)는 모든 레포에 토씨까지 동일하게 유지한다. 2~3부(매트릭스)는 각 조각이 자기 부분을 권위 있게 채우고, 다른 조각 부분은 참조 사본으로 들되 갱신 시점을 명시한다.

---

## 1부. 헌장 (모든 레포 동일 — 생태계의 방향)

거대 생태계는 어느 하나의 레포가 전부를 아우르지 않는다. 목적이 다른 조각들이 각자 독립 완성되어 가며 맞물려 하나의 전문 RA 시스템을 이룬다(덩치 때문에 의도적으로 분할 구축).

**공통 원칙**

1. **중앙 두뇌 없음** — 전체를 통제하는 단일 조정자를 두지 않는다. 판단은 분산하고, 집계는 규칙으로, 출력은 배관으로 처리한다.
2. **독립 완성** — 각 조각은 자기 목적으로 독립적으로 완성도를 높인다. 한 조각의 완성도가 생태계 전체 성능을 좌우한다.
3. **연결은 사람** — 조각 간 연계(언제·무엇을·어떻게 잇는가)는 전체 목적을 쥔 사람만 판단한다. 연계는 그 자체로 별도 프로젝트가 된다.
4. **단방향 의존** — 조각 간 의존은 방향이 명확해야 한다. 지식 토대는 업무 조각에 의존하지 않으며, 업무 조각이 지식 토대를 단방향 참조한다.
5. **지식 토대 공유, 중복 금지** — 공통 지식은 토대 조각(llm-wiki 등)을 통해 조회하며, 각 조각이 자체 RAG를 중복 생성하지 않는다.
6. **방향 일관** — 사본이 여러 레포에 흩어져도 이 헌장(방향)은 동일하게 유지한다. 상태(매트릭스)는 어긋날 수 있으나 갱신 시점 표기로 드러낸다.
7. **프랙탈 확장** — 같은 구조가 규모를 바꿔 반복된다(전문가:팀 = 팀:조직). 설계는 미리 전체를 그려 자리를 비워두고 구현만 단계적으로 한다. workspace는 조직 단위이며, 팀이 늘면 workspace가 늘어난다. 한 층에서 검증된 패턴(격리·분산판단·종합·투표·평가·단방향의존·사람최종결정)은 다음 층에 복제만 한다. 사람도 이 프랙탈의 일부로, 단위에 맞춰 할당된다(팀 담당 → 그룹 총괄 → 전체 책임).

**갱신 규율**

- **운영 중(자동화) 조각**: 매 스케줄 실행 시 자기 부분 매트릭스를 갱신한다(자동화 루프에 한 단계로 포함).
- **구현 중 조각**: 진행률 점검 시 자기 부분 매트릭스를 갱신한다.
- **상태 전환**: 조각이 구현→운영으로 넘어가면 갱신 방식도 점검형→스케줄형으로 전환한다(명세 매트릭스의 "갱신 방식" 칸 수정).
- 모든 항목은 "최종 갱신" 시점을 표기해 신선도를 드러낸다. 갱신은 사람이 각 프로젝트를 진행하며 직접 수행한다(에이전트 자동 갱신 금지 — 판단이 들어가는 칸은 사람이 쓴다).

## 4부. 연계 프로젝트 색인 (관계 매트릭스에서 파생 — 사람이 띄움)

조각이 충분히 완성되면 사람이 아래 연계를 별도 프로젝트로 판단·착수한다. 현재는 모두 미착수.

- RA 전문가 ← llm-wiki: 지식 조회 인터페이스 연결 (llm-wiki 인덱싱 충분 완성 후)
- RA 전문가 ← ra-project / MD-process: 규제지식·내부절차 참조 (조회 경로를 llm-wiki로 통합할지 별도로 둘지 결정 포함)
- RA 전문가 ↔ Regula: 업무 중 질의 / 실무경험의 지식 환류 (양방향, 먼 후속)
- 가상오피스 ← RA 전문가: 목업 데이터 계약으로 MVP 선개발 → 연동 시 데이터 소스만 목업→Honcho 교체. 계약 = 뼈대 활동 기록 출력 형태(conclusion·투표·매칭·3점평가). 뼈대는 시각화 존재를 모르고 운영 기록만 남김(단방향).

## 부록. 본 레포에서 채울 것

각 레포는 이 문서를 주입한 뒤, 2부에서 **자기 조각의 행**을 권위 있게 갱신하고, 3부에서 자기가 소비/제공하는 관계 칸을 갱신한다. 1부(헌장)는 수정하지 않는다(생태계 차원의 헌장 변경 시에만 전체 동기화).
