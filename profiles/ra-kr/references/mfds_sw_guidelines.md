# MFDS 소프트웨어 의료기기 허가·신고 가이드라인 핵심 요약

> 기준: 식품의약품안전처 「소프트웨어 의료기기 허가·신고·심사 가이드라인」(2023.12 개정)
> 참조: 의료기기법(법률 제19932호), 의료기기 허가·신고·심사 등에 관한 규정(식약처 고시)

---

## 1. 소프트웨어 의료기기 해당 여부 판단

### 의료기기 해당 기준 (의료기기법 제2조)

소프트웨어가 다음 **모든 조건**을 충족하면 의료기기:
- 질병 진단, 치료, 경감, 처치, 예방 목적
- 인체에 직접 또는 간접으로 영향
- 제조업자가 의료 목적을 표방

### 소프트웨어 유형별 분류

| 유형 | 정의 | 의료기기 해당 |
|------|------|------------|
| 독립형 소프트웨어(SaMD) | 하드웨어 없이 독자적으로 의료 기능 수행 | O |
| 임베디드 소프트웨어 | 의료기기 하드웨어 내장 소프트웨어 | 하드웨어와 함께 |
| 모바일 앱 | 스마트폰/태블릿 기반 의료 앱 | 의료 목적이면 O |
| 일반 건강관리 앱 | 단순 체중/운동 관리 등 | X (의료기기 불해당) |

### AI/ML 기반 소프트웨어

- 임상 의사결정을 지원하거나 진단 결과를 제공하는 경우 → 의료기기
- 계속 학습(Continuous Learning) 알고리즘: 잠금(Lock) 알고리즘 요건 적용
- 2022 AI 의료기기 허가 가이드라인 별도 적용

---

## 2. 등급 분류

### 분류 기준 (의료기기법 시행규칙 별표 1)

| 등급 | 위험도 | 심사 기관 | 허가/신고 구분 |
|------|--------|---------|------------|
| 1등급 | 잠재적 위험성 낮음 | 해당 없음 | 신고 |
| 2등급 | 잠재적 위험성 낮음(다소 높음) | 지정심사기관 | 인증 |
| 3등급 | 잠재적 위험성 높음 | 식품의약품안전처 | 허가 |
| 4등급 | 잠재적 위험성 매우 높음 | 식품의약품안전처 | 허가 |

### SaMD 등급 예시

- 1등급: 단순 체온 기록 앱, 관리 목적 소프트웨어
- 2등급: 심전도 리듬 분석(보조), 혈당 측정 보조 소프트웨어
- 3등급: CT/MRI 영상 AI 진단 보조, 방사선 치료 계획 소프트웨어
- 4등급: 자동 의사결정 진단 소프트웨어 (예: 자율 치료 결정)

---

## 3. 허가·인증·신고 절차

### 제출 서류 (소프트웨어 의료기기)

**필수 공통 서류:**
1. 허가(인증·신고) 신청서
2. 기술문서
   - 2-1. 의료기기 개요 (기능, 작동 원리, 적용 대상)
   - 2-2. 소프트웨어 설명서 (SDS — Software Description Specification)
   - 2-3. 안전·성능 규격 적합성 (인용 규격 목록 + 시험 결과)
   - 2-4. 임상시험 자료 또는 문헌 (3등급 이상)
3. GMP 적합성 인정서 또는 제조업 등록증
4. 라벨 및 사용설명서(IFU)

**소프트웨어 설명서(SDS) 필수 항목:**
- 소프트웨어 식별 (이름, 버전, 릴리즈 날짜)
- 개발 환경 (언어, OS, 프레임워크, 의존성)
- 소프트웨어 아키텍처 다이어그램
- 기능 유닛 분해 (Functional Units)
- SOUP(Software Of Unknown Provenance) 목록
- IEC 62304 소프트웨어 안전등급 (A/B/C)
- 소프트웨어 테스트 요약 (단위/통합/시스템 테스트)
- 사이버보안 대책 (네트워크 연결 기기만)

## 4. 핵심 인용 규격

| 규격 번호 | 내용 | 필수 여부 |
|---------|------|---------|
| IEC 62304 | 의료기기 소프트웨어 생명주기 프로세스 | 2등급 이상 |
| IEC 82304-1 | 독립형 소프트웨어(SaMD) 안전성 요구사항 | SaMD |
| ISO 14971 | 의료기기 리스크 관리 | 전 등급 |
| IEC 60601-1 | 전기·기계적 안전 (소프트웨어가 하드웨어 제어시) | 해당 시 |
| IEC 62443 | 네트워크 연결 의료기기 사이버보안 | 네트워크 연결 시 |

## 5. 인·허가 후 요건

### 변경허가·신고

- **변경허가 필요**: 소프트웨어 명칭, 의도된 사용 목적, 알고리즘 주요 변경
- **변경신고**: 버전 업그레이드 (기능 동일, 버그 수정)
- **자체 관리**: UI 변경, 성능 개선 (안전성 영향 없는 경우)

### GMP 유지

- 연 1회 이상 내부 감사
- 변경 관리 절차 문서화
- CAPA 기록 유지

### 시판 후 감시(PMS)

- 불량 의료기기 신고 의무 (중대한 부작용)
- 시판 후 임상 추적조사(PMCF) — 일부 제품

---

## 6. 수입품 특이사항

- 제조국 GMP 인증서 또는 동등 문서 필요
- ISO 13485 인증으로 국내 GMP 대체 가능 (협약에 따라)
- CE 마크 보유 시: 기술문서 일부 인정 (등가심사 가능성 있으나 별도 심사 필수)
- FDA 510(k) 보유 시: 동등성 인정 불가 — 별도 국내 허가 필요

---

## 7. 심사 시 주요 지적 사항 (참고)

1. SDS의 소프트웨어 안전등급 근거 불충분
2. SOUP 목록 누락 또는 최신화 미흡
3. 리스크 관리 파일에 잔류 위험 수용 근거 부재
4. 임상 문헌 검색 전략 및 기준 설명 불충분
5. IEC 62304 Level B/C에 해당하는 소프트웨어 단위 시험 결과 누락
