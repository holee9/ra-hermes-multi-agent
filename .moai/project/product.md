# RA Hermes 멀티 에이전트 — 제품 비전

## 제품 정의

**RA Hermes 멀티 에이전트**는 의료기기 인허가(Regulatory Affairs, RA) 업무를 자동화하고 학습하는 AI 멀티 에이전트 시스템입니다. Hermes Agent 기반 지역별 RA 전문가 에이전트(FDA 미국, MDR 유럽, MFDS 한국)가 이메일 분류·분석·업무 패키지 관리를 담당하며, 매 판단마다 사람의 피드백을 받아 자동으로 성장합니다.

## 핵심 가치 제안

**사람은 판단만 한다. 에이전트가 학습하고 성장한다.**

- **자동화**: 규제기관별 이메일 분류·WP 라우팅을 초(秒) 단위로 처리
- **지역별 맞춤화**: 미국(510(k), substantial equivalence), 유럽(MDR, Notified Body), 한국(KGMP, 국제 정합) 각 체계를 독립된 에이전트가 전담
- **누적 학습**: 매 판단 후 3점 평가(판단 근거·실제 결과·차이)를 저장, 다음 판단에 자동 반영
- **감사 기록**: 모든 분석·피드백·학습 과정이 Honcho 메모리 DB에 저장되어 규제감시 대응 시 근거로 활용

## 대상 사용자

**운영자**: 
- 지역별 RA 전문가(3명 기준)
- OpenProject 사안 담당자

**최종 판단자**:

- T3610, GX10, Raspberry Pi 담당 운영자

## 핵심 기능

### 1. 이메일 자동 분류 및 분석

n8n의 mail-triage 워크플로우:
- **본문 파싱**: 재전송 이메일의 원문·발신자·사안 ID 추출
- **규제권 라우팅**: FDA/MDR/MFDS 판정 → 해당 RA 에이전트로 자동 할당
- **RA 분석 호출**: Hermes 프로파일 API로 규제분석 실행
- **판단 결과**: 기존 WP 매칭 여부, 신규 WP 필요 여부, 신뢰도 점수

### 2. 업무 패키지(WP) 자동 관리

OpenProject와 Honcho 연동:
- **자동 매칭**: 분석 결과를 OpenProject 기존 WP(오픈/진행중/리뷰중)와 매칭
- **코멘트 추가**: 일치하는 WP에 RA 분석 코멘트 자동 추가
- **신규 생성**: 미매칭 시 자동으로 신규 WP 생성(담당자 할당 대기)
- **기록 저장**: 사안↔WP 매핑을 Honcho에 conclusion으로 기록(다음 판단 시 참고)

### 3. 다차원 학습 및 피드백

Honcho 메모리 백엔드 활용:
- **Warm-Start**: 판단 전에 Honcho에서 관련 이전 판단 컨텍스트 자동 조회
- **Delta 기록**: 사람 피드백(판단 근거·실제 결과·점수) → 3점 평가 저장
- **Deriver 활성화**: 백그라운드에서 사람 피드백을 자동으로 추론·정제해 다음 판단에 반영

### 4. 성장 지표 측정

GROWTH-5 인프라(매일 자동 계산):
- **정확도(Accuracy)**: 에이전트 WP 매칭 제안 대비 사람 승인률
- **신속도(Velocity)**: 분석 완료 시간(초 단위)
- **신뢰도(Confidence)**: 평균 분석 신뢰도 점수
- **학습량(Learning)**: 매일 누적된 평가 피드백 수
- **자동율(Automation)**: 사람 개입 없이 자동 처리된 사안의 비율

## 개입 경계 (Gate Rules)

**자동(Green)** — 에이전트 자율:
- RA 분석 실행
- 기존 WP 코멘트 추가
- 신규 WP 생성 제안

**승인 대기(Yellow)** — 사람 확정 필요:
- 상태 전이(예: "리뷰중" → "승인")
- n8n 워크플로우 수정
- 규제권 판정 불가

**사람 전용(Red)** — 영구 고정:
- WP 완료(Close) — 에이전트 절대 불가
- WP 재개(Reopen) — 에이전트 절대 불가
- 인프라 구조 변경 — 사람 승인 필수

## 비목표 (Non-Goals)

**이 시스템이 하지 않는 것:**

- 🚫 **지식 기반(RAG) 구축**: 별도 프로젝트(llm-wiki, ra-project, MD-process)에서 제공하는 참고 자료를 단방향으로만 읽음. 중복 RAG 금지.
- 🚫 **일반 NLP 모델 학습**: 직접 모델 재학습하지 않음. 프롬프트·컨텍스트·평가 피드백으로만 성장.
- 🚫 **중앙 조정자(통제 시스템)**: 최종 판단은 항상 사람. 에이전트는 제안만 함.
- 🚫 **다중 Workspace 자동 관리**: 업무·인프라 두 workspace는 격리된 도메인. 브릿지는 단방향.
