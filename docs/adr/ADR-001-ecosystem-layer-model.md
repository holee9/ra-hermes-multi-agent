## 배경

2026-06-05 세션에서 7개 RA 레포를 전수 분석한 결과, 각 레포의 역할과 상호 관계를 팩트 기반으로 정리하고 개발 집중 지점을 명확히 할 필요성이 확인됨.

---

## 결정

### 레이어 모델

허브-스포크 아님. 3-레이어 단방향 공급 구조:

```
[지식 레이어] — 독립 자율운영, 단방향 공급
  ra-project      ──┐
  MD-process      ──┼──▶ [에이전트 레이어]       [인터페이스 레이어]
  nas-llm(NAS)    ──┘    ra-hermes-multi-agent ←── ra-med-bot (Regula)
```

### 각 레포 위상

| 레포 | 위상 | 운영 방식 |
|---|---|---|
| **ra-hermes-multi-agent** | 에이전트 레이어 — **개발 주력** | 사람+에이전트, 이 레포에서 이슈 관리 |
| ra-project | 지식 레이어 (규제) | 자동화 자율운영, 개발자 개입 최소 |
| MD-process | 지식 레이어 (SOP) | 자동화 자율운영, 개발자 개입 최소 |
| nas-llm | 지식 파이프라인 | Windows NAS PC 독립 운영 |
| ra-med-bot (Regula) | 인터페이스 레이어 | Wave 3/4 진행 중, 병렬 독립 완성 |
| regula-eval-suite | 품질 검증 | 동결 — ra-med-bot Wave 3/4 완료 후 재개 |

### 크로스레포 이슈 관리 원칙

- **이 레포 (ra-hermes-multi-agent)**: 크로스레포 아키텍처 결정, 에이전트 구현, hermes-ra 흡수 작업
- **각 레포 자체**: 해당 레포 내부 작업 (MD-process SOP, ra-med-bot Wave 작업 등)
- **hermes-ra**: 아카이브 처리 후 read-only

---

## 근거

1. **ECOSYSTEM.md 헌장**: §1 원칙 1("중앙 두뇌 없음"), 원칙 4("단방향 의존")가 레이어 구조를 강제함
2. **ra-hermes-multi-agent가 개발 주력인 이유**:
   - ECOSYSTEM.md (전체 아키텍처 헌장)가 이 레포에 있음
   - 에이전트 로직 (Honcho, 프로파일, n8n 통합)이 이 레포에서 구현됨
   - 미션 "지구최강 RA 전문가"의 실행 주체 에이전트가 이 레포에 있음
   - 지식 레이어 레포들은 이미 완성(100%) — 개발 집중 불필요
